# 一、研究動機與背景
在國中時，我首次接觸到「舊手機工具箱」——一種整合局域網（LAN）應用的平台。 透過將手機連接至同一 Wi-Fi 網路，使用者可以實現檔案傳輸、遠端監控，甚至將手 機作為小型伺服器使用。這種「設備在同一網路下無縫協作」的體驗，引起了我對 LAN 技術應用的濃厚興趣。查閱相關文獻後，我發現現有研究大多集中於 LAN 的 理論模擬，缺乏實務環境下的測量數據。 

為填補這一缺口，本研究將透過系統化實驗，收集並分析 LAN 的運作表現，包括延遲、傳輸速率與穩定性等。研究結果不僅可驗證理論，亦有助於提供 LAN 應用設計與優化的參考依據。 

# 二、研究目標
1. 理解區域網路（LAN）的運作原理與應用。 
2. 透過 Ping 測量，探討區域網路在檔案傳輸延遲上相較廣域網路的優勢。
 
# 三、區域網路（**LAN**）

## （一）區域網路（**LAN**）的運作原理 
區域網路（Local Area Network, LAN）是指位於特定小範圍內的網路，例如家庭、辦公室或校園內的網路。LAN 主要功能是讓多台電腦或裝置能夠 互相連接並共享資源，如 檔案、印表機、網際網路連線等。  

在典型的 LAN 中，所有設備通常都連接到一個中心裝置 ——路由器。家庭環境多使用單一路由器，而較大的 LAN 可能會搭配網路交換器（Switch），以提升封包傳送效率和網路穩定性。LAN 的設備可以透過乙太網路纜線（Ethernet）或無線 Wi-Fi 連接。 

\## 乙太網路（Ethernet）：利用實體纜線進行資料傳輸，穩定且速度快。 

\## Wi-Fi：透過無線電波連接設備，使用方便，適合行動裝置或無線需求高的環境。

## （二）區域網路（LAN）的應用 
區域網路的核心功能是讓多台裝置彼此連接和共享資源。在家裡或辦公室，最直接 的用途就是檔案共享-使用者之間傳輸文件不必透過網際網路，速度更快也更安全， 同時印表機、掃描器等周邊設備也能透過網路共用。 

在企業或學校環境中，LAN 的應用更進一步。應用程式可集中安裝在伺服器上，使用者直接從網路執行。內部通訊服務如電子郵件、即時通訊或視訊會議，也仰賴 LAN 讓成員快速協作。 

其他領域也能看見LAN的影響，例如：校園內，學生與老師透過網路存取學習平台與研究資料庫；醫療機構則利用LAN連接工作站與醫療系統，讓醫護即時查看病歷與影像等資料。在工廠，自動化設備與監控系統透過LAN實現即時控制；企業內部的伺 服器叢集也藉由LAN共享運算資源，支援大型系統運作。 

# 四、實驗

## （一）實驗目的
本實驗旨在比較區域網路（LAN）與廣域網路（WAN）在網路延遲時間上的差異，探討不同網路架構對傳輸效能的影響。 

## （二）測試環境與方法概述
測試環境： 

1. 手機：Redmi Note 12 Turbo 
   作業系統：Xiaomi HyperOS 3.0.1.0（中國版 ROM） 

2. 電腦：Swift SF114-32 
   作業系統：Windows 11 

3. 路由器：TOTO LINK T8
   
4.廣域網路（WAN）測試節點： "ipv4Prefix": "34.174.0.0/16",  "service": "Google Cloud", "scope": "us-south1" 

測試方法： 

本實驗使用電腦作為測試主機，透過 Ping 指令測量不同網路環境下的延遲時間。 首先，測試對象包含兩種類型： 

（一）連接於同一區域網路（LAN）的手機裝置。
（二）位於外部網際網路的網站（WAN）。 

測試時，於電腦命令提示字元（CMD）中輸入 Ping 指令，分別對區域網路內 的手機 IP 位址與外部網站網址進行測試。系統會顯示每次封包往返所需時間，包含最小值、最大值與平均值。 

為了增加實驗數據的差異性與比較效果，本實驗選擇位於美國的 Google 伺服器 (IP:34.174.0.0)作為外部測試目標，使測試封包需經由跨國網際網路傳輸。由於傳輸 距離較長、經過節點較多，理論上延遲時間會明顯高於區域網路環境。

總結 

本實驗透過 20 次 Ping 測試，比較區域網路（LAN）與廣域網路（WAN）的延遲差異。 根據實驗數據統計，區域網路的平均延遲約為 40.2 ms，而外部網路（美國 Google 伺 服器）的平均延遲約為 215.9 ms，後者約為前者的 5 倍以上，顯示兩種網路環境在傳輸效率上具有明顯差異。 

從數據分布觀察可知，區域網路的延遲雖然大多落在較低範圍，但仍出現數次明顯偏高的數值（如 141 ms），推測可能與無線網路干擾、設備負載或瞬間封包擁塞有關。然而，即使出現波動，其整體平均值仍顯著低於廣域網路。 

相較之下，廣域網路延遲整體偏高，且數值分布較為集中，主要原因為資料需經由多個路由節點並跨越較長距離傳輸。由此可知，傳輸距離與網路架構複雜度，對延遲時間具有顯著影響。 

綜合實驗結果可驗證區域網路在回應速度上確實優於廣域網路。 

# 五、結論
本研究透過 Ping 測試，比較區域網路（LAN）與廣域網路（WAN）的延遲差異，並探討 LAN 的運作原理與應用。 

根據 20 次測試結果，區域網路的平均延遲約為 40 ms，而外部網路的平均延遲約為 216 ms，約為區域網路的 5 倍以上。由此可見，區域網路在回應速度上確實優於廣域網路。 

從實驗結果可以發現，LAN 的延遲整體較低，但仍偶爾出現較高數值 ，可能與無線 網路干擾或裝置狀態有關；而 WAN 的延遲則整體較高，主要原因為資料需經過較長 距離與多個網路節點傳輸。因此，網路的傳輸距離與架構會直接影響延遲。 本研究 成功驗證區域網路在延遲表現上具有優勢，也更理解不同網路環境對資料傳輸速度 的影響，達成研究目標。 

# 六、研究限制

本研究雖已透過實際測試比較區域網路（LAN）與廣域網路（WAN）的延遲差異，但仍 存在部分限制。首先，本實驗僅在單一時間進行測試，未考慮不同時段網路流量變化 對結果的影響。其次，本次測試使用無線網路（WiFi）環境，可能受到訊號干擾或裝 置狀態影響，使部分數據出現波動。此外，外部網路僅選擇單一位於美國的伺服器作 為測試對象，未涵蓋其他國家或不同類型網站，因此結果仍有侷限性。 

未來若能增加測試次數、不同時間段測量，或比較有線與無線網路環境，將能使研究 結果更加完整與準確。
